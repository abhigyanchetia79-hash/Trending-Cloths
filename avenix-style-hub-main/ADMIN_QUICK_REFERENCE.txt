🎛️  ADMIN VISIBILITY TOGGLE
═════════════════════════════════════════════════════════════════════════════

LOCATION:
  → Admin Panel → Settings tab → "Admin Login Visibility"

WHAT IT DOES:

  🟢 TOGGLE = ON (VISIBLE)
     ✓ Admin login link shows in navbar & mobile menu
     ✓ Non-admins can see and click admin login
     ✓ Regular navigation to /admin/auth works

  🔴 TOGGLE = OFF (HIDDEN)
     ✗ Admin login link REMOVED from navbar & mobile menu
     ✗ Non-admins cannot find admin login option
     ✓ Direct URL access /admin/auth still works (for admins)
     ✓ Admin can access via Shield icon or direct URL

STORAGE:
  ♻️  Stored in localStorage (immediate)
  ♻️  Synced to Supabase admin_settings table (persistent)

🐛 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: Admin link not showing in navbar
Solution: 
  1. Verify you're logged in as admin
  2. Check if toggle is ON in Settings
  3. Clear browser cache: Ctrl+Shift+Delete
  4. Reset: localStorage.removeItem("showAdminLogin")

Problem: Can't create admin account
Solution:
  1. Check secret code: exactly "TRENDING2026"
  2. Look at console (F12) for error details
  3. Verify email is unique
  4. Password must be 6+ characters

Problem: Settings not saving
Solution:
  1. Check browser console for database errors
  2. Verify Supabase connection in .env file
  3. Run the migration again
  4. Check admin_settings table exists in Supabase

Problem: Can't access /admin dashboard
Solution:
  1. Make sure you're logged in as admin
  2. Check user_roles table has your user_id with role='admin'
  3. Refresh page (Ctrl+R or Cmd+R)
  4. Check browser console for errors
